# ChatInsight.Api — ROADMAP

План развития от текущего backend'а к продукту из `1.docx`.
Статус по пунктам — в `PROJECT_STATUS.md`.

---

## Сейчас (готово)

Backend-аналитика поверх Telegram Export: импорт, 9 аналитических модулей на
едином `ChatAnalysisContext`, сводный отчёт, Swagger. Всё в памяти, на один запрос.

---

## Этап 1 — Достроить MVP

Цель: из «считалки метрик за запрос» сделать продукт, который хранит данные и
отдаёт человеку отчёт.

1. **База данных.**
   - EF Core + Npgsql, `ChatInsightDbContext` в `Data/`.
   - Сущности в `Models/Domain/`: `Chat`, `Message`, `Participant`, `AnalysisResult`.
   - Миграции, строка подключения через User Secrets (не в `appsettings.json`).
2. **Сохраняемый импорт.**
   - `/api/import/telegram` пишет чат в БД и возвращает `chatId`.
   - Остальные эндпоинты принимают `chatId` вместо повторной загрузки файла.
   - Кэшировать `ChatAnalysisContext` по `chatId`.
3. **Экспорт отчёта в файл.**
   - PDF (пункт MVP из идеи), затем HTML и Markdown.
   - Складывать в `Reports/`, отдавать ссылкой/файлом.
4. **Полировка API.**
   - Консистентные коды ответов, единый формат ошибки.
   - Лимит размера загрузки, антифорджери при необходимости.

---

## Этап 2 — AI и качество анализа

Цель: заменить словари на модель и выдавать «инсайты», а не только числа.

5. **AI Engine на Ollama.**
   - Локальные модели (Qwen / Gemma / Llama) через HTTP.
   - Эмоции, темы, сводка-резюме переписки текстом.
   - `AI Insight Engine`: ключевые события, переломные моменты, охлаждение/сближение.
6. **Расширить Relationship.**
   - Достроить `InitiativeBalance` и `ResponseBalance`.
   - Свести в индексы близости / доверия / вовлечённости.
7. **pgvector.**
   - Эмбеддинги сообщений, семантический поиск и кластеризация тем.

---

## Этап 3 — Продукт (v2/v3 из идеи)

8. **Life Timeline Engine** — жизненная хронология по переписке, периоды, изменения личности.
9. **Frontend** — React + TS + Tailwind, графики, загрузка, просмотр отчётов.
10. **Мультиплатформенность** — Discord / WhatsApp / VK экспорт-парсеры.
11. **Групповой и корпоративный анализ**, публичный API, SaaS.

---

## Принцип очередности

Сначала **хранение → отчёт-файл → AI**. Фронт можно начинать параллельно после
Этапа 1 (когда появится стабильный `chatId`-контракт). Мультиплатформа — после
того, как пайплайн отлажен на Telegram: добавление источника = новый парсер,
который маппится в те же `Message`/`ChatAnalysisContext`.
