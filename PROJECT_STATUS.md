# ChatInsight — статус проекта

Платформа анализа переписок: backend (**ASP.NET Core .NET 9** + PostgreSQL + Ollama)
и frontend (**React + Vite + Tailwind**). Идея — в `1.docx`.

---

## Сверка с идеей (MVP 1.0) — закрыт полностью

| Пункт MVP | Статус |
|---|---|
| Импорт Telegram Export | ✅ (+ дозагрузка) |
| Сохранение сообщений | ✅ PostgreSQL + EF Core |
| Анализ активности | ✅ |
| Анализ скорости ответов | ✅ |
| Графики | ✅ Recharts |
| Базовый AI-отчёт | ✅ Ollama / llama3.1 |
| Экспорт PDF | ✅ QuestPDF (+ AI-анализ и портреты внутри) |

Сверх MVP: темы, эмоции, инициатива, таймлайн, **баланс отношений**, сравнение
периодов, **AI-портреты**, веб-интерфейс.

> `net9.0` вместо `.NET 10`; `pgvector` пока не подключён (v2.0).

## Что работает ✅

### Backend
- [x] Web API (.NET 9), Swagger, PostgreSQL + EF Core, Docker Compose
- [x] Импорт с дозагрузкой (по `SourceId`/`TelegramId`), сброс кэша AI
- [x] 9 модулей аналитики на едином `ChatAnalysisContext`
- [x] **Баланс отношений** (активность/инициатива/ответы) — в отчёте
- [x] Сравнение периодов
- [x] AI-инсайты и AI-портреты (Ollama, JSON Schema, кэш в БД)
- [x] **PDF с AI-анализом и портретами участников** (`?ai=true`)

### Frontend
- [x] Загрузка, список чатов, страница анализа
- [x] Графики (часы, авторы), блок баланса отношений
- [x] AI-блоки по кнопке (инсайты, портреты, сравнение), скачивание PDF

---

## Эндпоинты

| Маршрут | Метод | Что делает |
|---|---|---|
| `/api/import/telegram` | POST | импорт/дозагрузка |
| `/api/chats` · `/{id}` | GET | список / метаданные |
| `/api/chats/{id}/report` · `.pdf` | GET | отчёт JSON / PDF (`?ai=true` — с AI и портретами) |
| `/api/chats/{id}/insights` | GET | AI-выводы (кэш, `?refresh`) |
| `/api/chats/{id}/personality` | GET | AI-портреты (кэш, `?refresh`) |
| `/api/chats/{id}/compare` | GET | сравнение периодов (`?splitDate`) |

---

## Хранение

PostgreSQL, EF Core. Таблицы: **Chats** (+`SourceId`), **Messages**,
**Insights** (1 на чат), **Personalities** (1 на чат+участник).
