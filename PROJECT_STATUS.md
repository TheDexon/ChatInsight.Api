# ChatInsight — статус проекта

Платформа анализа переписок: backend (**ASP.NET Core .NET 9** + PostgreSQL + Ollama)
и frontend (**React + Vite + Tailwind**). Идея — в `1.docx`.

**MVP 1.0 — закрыт полностью** (импорт, хранение, активность, скорость ответов,
графики, AI-отчёт, PDF). **Версия 2.0 закрыта**: AI-хронология, эволюция личности, семантический поиск на
pgvector и **кластеризация тем по смыслу** (k-means над эмбеддингами).

## Хранение

PostgreSQL, EF Core. Таблицы: **Chats** (+`SourceId`), **Messages**, **Insights**,
**Personalities**, **LifeTimelines** (кэш хронологии), **AiJobs** (фоновые задачи).

---

## Единый фундамент: выжимки по периодам (Digest Engine)

Весь чат режется на куски (~250 сообщений), по каждому AI-выжимка сохраняется в БД
(`PeriodDigests`) — строится ОДИН раз. Поверх выжимок (100% охват, не выборка) строятся:
- **AI-анализ** (инсайты), **Хронология жизни** (вехи с категориями), **Rollup** (итог+таймлайн).
- Первый запуск любого из них строит выжимки с прогрессом «N/M периодов», дальше — мгновенно.
- Портреты и эволюция остаются на выборке (стиль/голос лучше читать по сырым сообщениям).

## Полный анализ по периодам (Rollup Engine)

Двухфазный анализ ВСЕЙ переписки вместо выборки в 1%:
- чат режется на куски (~250 сообщений), по каждому AI делает выжимку
  (о чём, настроение, события) — фаза 1;
- все выжимки собираются в единую хронологию + итог — фаза 2.
- Прогресс «обработано N/M периодов» (поле `AiJob.Progress`).
- Учитывает медиа без текста (стикеры/гс/фото считаются по периодам),
  игнорирует мусор, помнит про иронию. Кэш — `Rollups`.

## Качество анализа (допилка)

- **Фильтр мусора** (`MeaningfulTextFilter`): рандом с клавиатуры, повторы букв,
  эмодзи и слишком короткие сообщения не идут в эмбеддинги/кластеры (~10% шума).
  Повторное построение индекса само удаляет ранее попавший мусор.
- **Подсказка про иронию** (`AiPrompts.IronyNote`) во всех AI-промптах: модель не
  принимает шутки/сарказм/мат за реальную агрессию и события.
- **Пересчёт**: у каждого AI-блока кнопка «Пересчитать» (async `?refresh=true` →
  сброс кэша → свежий результат).

## AI-слой

Ollama локально: `llama3.1:8b` (анализ) + `nomic-embed-text` (эмбеддинги). Structured output по JSON Schema, кэш всех
тяжёлых результатов в БД. Тяжёлые вызовы идут через фоновую очередь; фронт опрашивает
статус. Воркер сериализует результат в camelCase (совпадает с фронтом).
